# 快速开始

5 分钟快速配置 NoWakeLock，开始优化设备电池续航。

## 第 1 步：验证安装

打开 NoWakeLock，进入"模块检查"页面：

- ✅ 所有项目显示绿色 = 安装成功
- ❌ 任何红色项目 = 查看[安装指南](installation.md)

【需要截图：模块检查成功界面】

## 第 2 步：查看应用列表

1. 点击底部"应用"标签
2. 浏览已安装的应用列表
3. 注意右侧的统计数字

【需要截图：应用列表界面】

**列表信息**：
- 应用名称和图标
- WakeLock/Alarm/Service 统计
- 最后活动时间

## 第 3 步：分析耗电问题

⚠️ **重要**：配置任何规则前，必须先确认设备确实存在电池问题。

### 使用问题分析工具
- 查看 [问题分析指南](problem-analysis.md) 了解完整的分析流程
- 使用 BetterBatteryStats 或 NoWakeLock 内置统计确认问题
- 只对确认有问题的 WakeLock 进行配置

### 快速识别问题应用
1. 点击"Wakelocks"标签
2. 关注"计数"和"时长"较高的项目
3. 这些是主要耗电来源

【需要截图：WakeLock 列表】

## 第 4 步：设置基础规则

### 配置问题 WakeLock
1. 点击异常的 WakeLock 项目
2. 选择处理方式：
   - **限制** - 超时释放（推荐新手）
   - **拦截** - 完全阻止（谨慎使用）

【需要截图：设置规则界面】

## 观察效果

### 24 小时后检查
1. **应用界面** - 查看拦截效果
2. **应用功能** - 确认重要功能正常
3. **电池使用** - 系统电池统计

### 关键指标
- 拦截次数增加 ✅
- 应用推送正常 ✅  
- 电池续航改善 ✅

【需要截图：统计图表】

## 常见问题

### 某应用无法接收通知
**解决方案**：
1. 找到该应用的推送相关 WakeLock
2. 改为"允许"模式
3. 观察几小时

### 设置后没有效果
**检查项目**：
1. 模块是否正常工作
2. 规则是否正确应用
3. 应用是否重启

### 设备变慢或卡顿
**立即处理**：
1. 暂时禁用所有规则
2. 逐步恢复重要应用
3. 避免拦截系统关键服务
